# Code Style Guidelines

To keep the SecuRock codebase clean, readable, and maintainable, we adhere to the following coding styles:

## Backend (Python)
- Follow PEP 8 guidelines.
- Use explicit type hints for all function arguments and return types.
- Format codebase using black/autopep8 style guide format.
- Document classes and complex logic methods using standard docstrings.

## Frontend (TypeScript / React)
- Use standard camelCase for variables/functions, PascalCase for components/types.
- Statically type components using TypeScript interfaces or types.
- Adhere to clean, reusable components philosophy.

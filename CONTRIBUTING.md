# Contributing to SecuRock SOC

Thank you for your interest in contributing to SecuRock! This document outlines the guidelines and procedures for contributing to our autonomous SOCaaS platform.

## Code of Conduct
Please be respectful and helpful in all your interactions within this project.

## Getting Started
1. Fork and clone the repository.
2. Follow the setup instructions in the [README.md](README.md) file.
3. Make sure tests run successfully before creating any pull requests.

## How to Run Tests
Inside the `backend` directory, run:
```bash
python -m pytest
```

## Creating Pull Requests
- Keep changes concise and focused on a single feature or bug fix.
- Ensure all new features are covered by tests.
- Format code following Python PEP 8 style standards.
